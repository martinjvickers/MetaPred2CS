Result

Depends on the parameters which selected by user, reaching result page can be time consuming. For this reason, at the result page: employed files and used parameters by server during the calculation will be seen beside MetaPred2CS result as tables.
At result table of server, users will see 3 columns which are ID of query proteins (that will be composed by the locul tags of query proteins), class of result and the prediction score. Here in class section only 2 different variables can be seen and these variables are -1 or +1. -1 is representing that the query proteins are not positively interacting so user will see prediction score as 0 for that query. However if the query classified as +1 then user will be able to see a prediction score. Scoring range of MetaPred2cs is 0-1.
Based on prediction score, predictions can be classified as high confidence, medium confidence or low confidence. Score ranges of these classes are defined as 0.999-0.700, 0.699-0.400 and 0.399-0.000, respectively.

Important Points

1) MetaPred2CS is trained on single domain two-component system protein-protein interactions. For this reason, to be able to perform predictions for hybrid two-component system proteins, users need to work at domain level. In other words, protein sequences of domains of hybrid proteins should be given separately instead of complete protein sequence.

2) During the MSA reducing step of i2h and MT methods, if required minimum number of common species not ensured calculation will be terminated. Under this circumstance, users should lower the min matches limit or higher the b_value and restart the job.
